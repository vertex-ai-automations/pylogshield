How should the enhanced log viewer work?

The current viewer is a Rich table you call from the CLI. Three directions to take it further — pick the one that fits how your users actually work.

A

Interactive TUI (Textual)

A full terminal UI — keyboard navigation, live search bar, filter panel, paged results, export. Runs in the same terminal, no browser required. Built with Textual (same author as Rich, already in your dependency tree).

Pros

  • Works anywhere — SSH, CI, headless servers
  • Keyboard-first, fast to navigate
  • Feels like htop or lazygit

Cons

  • Adds Textual as a dependency
  • Mouse support varies by terminal emulator
B

Enhanced CLI Commands

Keep the same pylogshield view / follow pattern but add new subcommands: search, stats, export. No new UI paradigm — just more flags and commands piped to stdout.

Pros

  • Zero new dependencies
  • Scriptable — plays well with grep, awk, jq
  • Simple mental model

Cons

  • No interactive experience
  • Analysis requires multiple separate commands
C

Local Web Dashboard

A pylogshield dashboard --file app.log command that spins up a local web server and opens a browser dashboard. Full search, charts, filters, export — all in HTML/JS.

Pros

  • Richest UI — charts, date pickers, export buttons
  • Shareable via screenshot or HTML export

Cons

  • Requires a browser — won't work headless
  • Significantly more code to build and maintain
  • Adds a web server dependency
